## Button groups

If buttons are related, they can be grouped with a `ButtonGroup`.

<ComponentShowcase>
	<ButtonGroup>
		<Button>First</Button>
		<Button>Second</Button>
		<Button>Third</Button>
	</ButtonGroup>
</ComponentShowcase>

```jsx
// highlight-next-line
<ButtonGroup>
	<Button>First</Button>
	<Button>Second</Button>
	<Button>Third</Button>
// highlight-next-line
</ButtonGroup>
```

Button groups improve focus behavior - instead of having to `Tab` through each button,
the group is treated as a single focusable element that can be navigated with left/right arrows.

### Vertical group

<ComponentShowcase>
	<ButtonGroup vertical>
		<Button>First</Button>
		<Button>Second</Button>
		<Button>Third</Button>
	</ButtonGroup>
</ComponentShowcase>

In this case, the focus navigation is handled with up/down arrows.

```jsx
<ButtonGroup
	// highlight-next-line
	vertical
>
	<Button>First</Button>
	<Button>Second</Button>
	<Button>Third</Button>
</ButtonGroup>
```

:::note
By default, vertical button groups take up the full width of their container.
:::
